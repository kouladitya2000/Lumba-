Great — since you already have ASE deployment via Terraform, below is a clean, production-ready Terraform module structure plus a CI/CD pipeline that:
	•	Takes the ASE resource ID as input
	•	Deploys:
	•	App Service Plan (isolated, bound to ASE)
	•	Function App
	•	Application Insights
	•	Storage Account
	•	RBAC
	•	App settings
	•	Keeps things modular, readable, and pipeline-friendly

I’ll assume Azure DevOps Pipelines and Terraform ≥ 1.5. This works just as well for GitHub Actions with minor syntax changes.

🚀 Azure DevOps Pipeline – pipelines/azure-pipelines.yml

trigger:
- main

parameters:
- name: aseId
  type: string

variables:
  terraformVersion: '1.6.6'

stages:
- stage: Deploy
  jobs:
  - job: Terraform
    pool:
      vmImage: 'ubuntu-latest'

    steps:
    - task: TerraformInstaller@1
      inputs:
        terraformVersion: $(terraformVersion)

    - task: TerraformTaskV4@4
      displayName: Terraform Init
      inputs:
        provider: 'azurerm'
        command: 'init'
        backendServiceArm: 'AzureServiceConnection'
        backendAzureRmResourceGroupName: 'tfstate-rg'
        backendAzureRmStorageAccountName: 'tfstatestore'
        backendAzureRmContainerName: 'tfstate'
        backendAzureRmKey: 'function-ase.tfstate'

    - task: TerraformTaskV4@4
      displayName: Terraform Plan
      inputs:
        provider: 'azurerm'
        command: 'plan'
        environmentServiceNameAzureRM: 'AzureServiceConnection'
        commandOptions: >
          -var "ase_id=${{ parameters.aseId }}"
          -var-file="env/dev.tfvars"

    - task: TerraformTaskV4@4
      displayName: Terraform Apply
      inputs:
        provider: 'azurerm'
        command: 'apply'
        environmentServiceNameAzureRM: 'AzureServiceConnection'
        commandOptions: >
          -auto-approve
          -var "ase_id=${{ parameters.aseId }}"
          -var-file="env/dev.tfvars"
